# Fraud Detector

A full-stack credit-card fraud detection web application.

| Layer     | Technology                              |
|-----------|-----------------------------------------|
| Backend   | Python · FastAPI · scikit-learn         |
| ML Model  | Isolation Forest (joblib)               |
| Frontend  | React 18 · Vite                         |
| Deploy    | Backend → Render · Frontend → Vercel    |

---

## Local Development

### 1 — Download the Dataset

Download **creditcard.csv** from Kaggle:
<https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud>

Place the file inside `backend/`:
```
backend/creditcard.csv
```

### 2 — Backend Setup

```bash
cd backend
python -m venv .venv

# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

pip install -r requirements.txt
```

### 3 — Train the Model

```bash
# still inside backend/
python train.py
```

This produces `backend/model.pkl` and prints evaluation metrics:
- Precision, Recall, F1-Score
- AUC-ROC
- Confusion matrix

### 4 — Start the API Server

```bash
uvicorn main:app --reload --port 8000
```

API docs available at <http://localhost:8000/docs>

### 5 — Frontend Setup & Dev Server

```bash
cd frontend
npm install
npm run dev
```

Open <http://localhost:5173> in your browser.

## API Reference

### `POST /predict`

**Request body**
```json
{
  "amount":   149.62,
  "time":     406,
  "features": [0.0, 0.0, ..., 0.0]   // 28 floats (V1–V28), optional
}
```

**Response**
```json
{
  "result":     "Legitimate",
  "confidence": 0.87
}
```

`result` is either `"Fraud"` or `"Legitimate"`.
`confidence` is a value between `0.0` and `1.0`.

## Deployment

### Backend → Render (free tier)

1. Push the `backend/` folder to a GitHub repo.
2. Create a new **Web Service** on [Render](https://render.com).
3. Set the **Build command**:
   ```
   pip install -r requirements.txt
   ```
4. Set the **Start command**:
   ```
   uvicorn main:app --host 0.0.0.0 --port 8000
   ```
5. Add the environment variable:
   ```
   MODEL_PATH=model.pkl
   ```
6. Upload / commit `model.pkl` alongside your code, or use a Render disk.

> **Note:** The free tier spins down after inactivity — the first request may take ~30 s.

---

### Frontend → Vercel (free tier)

1. Push the `frontend/` folder to a GitHub repo (or a sub-directory of a monorepo).
2. Import the project on [Vercel](https://vercel.com).
3. Set the **Framework Preset** to **Vite**.
4. Add an environment variable in the Vercel dashboard:
   ```
   VITE_API_URL=https://your-render-service.onrender.com
   ```
5. Deploy — Vercel picks up `VITE_API_URL` at build time and embeds it.

The frontend already reads this variable:
```js
const API_BASE = import.meta.env.VITE_API_URL ?? ''
```

No code changes needed; the `??''` fallback keeps local dev working via the
Vite proxy defined in `vite.config.js`.

## How It Works

1. **Isolation Forest** is an unsupervised anomaly-detection algorithm.
   It isolates observations by randomly selecting a feature and a split value.
   Anomalies (fraud) are isolated faster → they have shorter average path lengths.

2. **`train.py`** scales `Amount` and `Time` with `StandardScaler`, combines
   them with the PCA-transformed `V1–V28` features from the Kaggle dataset, and
   fits an `IsolationForest`.  The scaler and model are bundled into `model.pkl`.

3. **`main.py`** loads `model.pkl` on startup, applies the same scaling to
   incoming requests, calls `model.predict()`, converts the raw anomaly score
   to a confidence value via a sigmoid function, and returns the result.

4. **`App.jsx`** sends a `POST /predict` request, renders a green card for
   legitimate transactions and a red card for fraud, with a circular confidence
   gauge.

---

## Notes

- `V1–V28` in the Kaggle dataset are the result of PCA transformation to
  protect cardholder privacy.  When using the UI without those features, they
  default to `0.0`, which is the mean of each PCA component (valid input).
- The `contamination` hyper-parameter in `train.py` controls the model's
  sensitivity.  Increase it to catch more fraud (higher recall, lower
  precision); decrease it to reduce false positives.
