# FraudGuard — Credit Card Fraud Detection Dashboard

A full-stack fraud detection web application with a real-time dashboard, XGBoost ML model, and AI-powered explanations.

| Layer          | Technology                                          |
|----------------|-----------------------------------------------------|
| Backend        | Python · FastAPI · SQLAlchemy · SQLite              |
| ML Model       | XGBoost (supervised) · scikit-learn · joblib        |
| AI Explanation | LangChain · OpenAI GPT-4o-mini                      |
| Frontend       | React 18 · Vite · Recharts · React Router           |
| Container      | Docker · Docker Compose                             |
| Deploy         | Backend → Render · Frontend → Vercel                |

## Features

- **Real-time Dashboard** — 4 KPI cards (Total Transactions, Fraud Detected, PR-AUC Score, Pending Review)
- **Live Transaction Table** — filter by All / Fraud / Review / Safe, auto-refreshes every 30 seconds
- **Live Alerts Panel** — high-confidence fraud flagged instantly
- **Fraud by Hour Chart** — bar chart showing hourly fraud patterns (last 24 hours)
- **Full Transaction History** — paginated table with timestamps and fraud scores
- **Model Performance Page** — PR-AUC, precision, recall, F1, confusion matrix, feature importance chart
- **Reports Page** — pie chart + hourly activity breakdown
- **AI Explanations** — LangChain + GPT explains WHY each transaction was flagged
- **Live Simulation** — new transactions added automatically every 30 seconds
- **Docker Support** — run the full stack with one command

## Local Development

### 1 — Download the Dataset

Download **creditcard.csv** from Kaggle:
https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

Place it inside `backend/`:
```
backend/creditcard.csv
```

### 2 — Backend Setup

```bash
cd backend

# Create and activate virtual environment (use Python 3.11–3.13)
py -3.13 -m venv .venv
.venv\Scripts\Activate.ps1      # Windows
# source .venv/bin/activate     # macOS / Linux

pip install -r requirements.txt
```

### 3 — Train the Model

```bash
python train.py
# OR without the dataset:
python train.py --demo
```

Prints evaluation metrics on completion:
- PR-AUC (main metric for imbalanced data)
- Precision, Recall, F1-Score
- Confusion matrix
- Optimal decision threshold

Produces `model.pkl` — the trained model bundle.

### 4 — Configure AI Explanations (Optional)

```bash
copy .env.example .env
```

Edit `.env` and add your OpenAI API key:
```
OPENAI_API_KEY=sk-your-key-here
```

Get a key from: https://platform.openai.com/api-keys

### 5 — Start the Backend

```bash
python -m uvicorn main:app --reload --port 8000
```

On first run, seeds ~600 historical transactions from `creditcard.csv` into SQLite.

### 6 — Start the Frontend

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173

## How the ML Works

### Why XGBoost?
XGBoost is a **supervised** algorithm — it learns from labelled examples (fraud vs. legitimate). It builds 300 decision trees sequentially, each correcting the errors of the previous one.

### Class Imbalance
The dataset has only **0.17% fraud** (492 out of 284,807 transactions). A naive model would just predict "Safe" for everything and be 99.83% accurate — useless.

Fix: `scale_pos_weight = n_legit / n_fraud ≈ 578` — tells XGBoost to treat each fraud case as if it were 578 legitimate cases.

### Why PR-AUC instead of Accuracy?
PR-AUC (Precision-Recall Area Under Curve) measures performance specifically on the minority class (fraud). It is the standard metric for heavily imbalanced classification problems.

### Decision Thresholds
```
XGBoost outputs a fraud probability (0.0 → 1.0):
  score >= 0.70  →  Fraud
  score >= 0.40  →  Review  (manual inspection recommended)
  score <  0.40  →  Safe
```

The threshold is automatically optimised during training to maximise F1 score.

### Features
- **V1–V28**: PCA-transformed features (original features hidden for privacy)
- **Amount**: Scaled with StandardScaler
- **Time**: Seconds since first transaction, scaled with StandardScaler

## Deployment

### Backend → Render

1. Push to GitHub
2. Create a **Web Service** on [Render](https://render.com)
3. Build command: `pip install -r requirements.txt`
4. Start command: `uvicorn main:app --host 0.0.0.0 --port 8000`
5. Environment variables:
   ```
   MODEL_PATH=model.pkl
   OPENAI_API_KEY=sk-your-key-here
   ```

### Frontend → Vercel

1. Import the repo on [Vercel](https://vercel.com)
2. Framework preset: **Vite**
3. Environment variable:
   ```
   VITE_API_URL=https://your-render-service.onrender.com
   ```
